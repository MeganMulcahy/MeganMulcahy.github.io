Lox Language Interpreter

The LOX interpreter I've developed adheres to the principles outlined in Chapters 4-10 of the book "Crafting Interpreters." It encompasses scanning, representation, parsing, evaluation, and control flow within the LOX programming language. This interpreter serves as my first completed project for this class, laying a foundational understanding that I plan to leverage in future interpreter endeavors. Inspired by the Java implementation in the book, I opted for a C# implementation to capitalize on its familiarity, especially for those transitioning from Java. The project's backbone lies in its organization through the Microsoft.NET.Sdk, defining an executable output for the net48 framework. The CraftingInterpreters.Lox.csproj file is meticulously structured, incorporating vital references to System, System.Core, System.Data, and System.Xml, forming a robust foundation for the interpreter's core functionality.

Implementation & Progression

The LOX interpreter's foundation was laid by creating a C# project in VSCode on a MAC, complemented with a GitHub repository for version control. Progress was diligently tracked after each chapter, saving project files to document my journey. I adopted a methodical approach to implementation, utilizing ChatGPT as a resource to streamline translations and enhance error handling. This pragmatic strategy reflected a deep understanding of the work, keeping the interpreter in sync with current technology trends.

In the "LOX Interpreter" folder, I've submitted key files, including Interpreter.cs, LoxCallable.cs, LoxFunction.cs, Parser.cs, and Stmt.cs, showcasing the core logic for executing LOX code, handling callable objects, functions, parsing, and managing statements. The Makefile streamlines project compilation, and tester.lox serves as a script for testing interpreter functionality.

Compilation and Running

For you as a user, with all the files downlaoded, in the terminal type in Make and the test file will automatically run.

To compile and run the LOX interpreter, I primarily leverage Mono for compilation and Bash for execution. Alternatively, JetBrains Rider or Visual Studio are viable alternatives. Bash, inherent in Linux and MacOS, is used for running the tester script. For Windows users, installing Cygwin enables the execution of bash scripts.

Executing dotnet build and dotnet run within the LOX Interpreter directory compiles and runs the interpreter. The Makefile automates these tasks and executes the test file, tester.lox. This approach ensures a seamless integration of compilation, execution, and testing, providing an efficient workflow.

Testing

The testing suite for my LOX Interpreter covers a diverse range of language features, ensuring a comprehensive assessment of the interpreter's performance. It includes tests from the book "Crafting Interpreters" to verify the correctness of the interpreter against the Java implementation. Rigorous testing validates the correct handling of variable values, reassignments, scoping rules, and variable isolation.

Expressions, control flow structures, logical operators, arithmetic operations, and complex expressions underwent meticulous testing. Features such as closures, functions, equality comparisons, and iterations were scrutinized to ensure accuracy and robustness in handling diverse scenarios.

The testing suite serves as a crucial tool for error identification, modification, and optimization of functionality. It addresses potential gaps in the book's completeness and showcases the interpreter's ability to handle a wide array of language features reliably. My commitment to rigorous testing reflects my dedication to crafting a functional and dependable LOX interpreter.
