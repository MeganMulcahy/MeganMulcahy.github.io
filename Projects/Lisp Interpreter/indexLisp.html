Lisp Language Interpreter

Building this Lisp interpreter has been an enriching journey, starting with aligning it with the Lox interpreter and translating those concepts into Lisp. The interpreter is implemented in the C# programming language, emphasizing practicality and clarity in language implementation. The grammar, inspired by Lisp conventions, encompasses atoms, symbols, numbers, strings, and various forms of lists, providing a solid foundation for language interpretation. The interpreter employs a list of S-expressions (SExpr) and a dictionary (vars) for variable storage. Noteworthy features include support for Lisp operations, variable assignments, function definitions, and conditional statements.

Implementation & Progression

The project was initiated by creating a C# project in VSCode on a MAC, utilizing GitHub for version control. I incorporated insights from ChatGPT for translation and error handling. The implementation follows a defined grammar, encapsulated within the Parser class, with comprehensive methods for parsing atoms, null expressions, and various forms of lists. The project structure, including the Lisp.csproj file, reflects a well-organized approach. The GitHub repository showcases a continuous progression of pushes and changes, highlighting a commitment to staying current with technology trends.

Compilation and Running

To compile and run the Lisp interpreter, Mono and Bash are primarily leveraged for compilation and execution, respectively. JetBrains Rider or Visual Studio are viable alternatives. The Makefile streamlines the compilation process, ensuring an efficient workflow. Running the tester script provides a robust method for testing interpreter functionality, with screenshots of testing and output provided in a dedicated folder.

Testing

Tests cover a range of functions, showcasing arithmetic and comparison operations, list manipulation, type-checking, logical operators, and user-defined functions through DEFINE. The interpreter handles various mathematical functions, equality checks, list manipulation operations like CONS, CAR, and CDR, and dynamic type-checking functions. While the interpreter demonstrates robust functionality, it's acknowledged that there are limitations, such as challenges in handling certain list definitions, which will be addressed in future iterations.

This Lisp interpreter represents a significant step forward, showcasing a comprehensive understanding of Lisp principles, effective implementation, and a commitment to continuous improvement. The journey involved aligning theoretical knowledge with practical application, utilizing both traditional and modern tools for a holistic development process. The interpreter stands as a testament to dedication and the pursuit of excellence in the realm of language interpretation.
